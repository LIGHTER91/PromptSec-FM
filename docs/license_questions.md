These concise questions are prepared for maintainers; Phase 3.4 does not contact them
automatically. This list is not legal advice.

- BIPIA: Are `benchmark/text_attack_*.json` and `benchmark/code_attack_*.json`
  redistributable as a derived research dataset, and which third-party contexts are
  included in each file?
- BIPIA: Which exact records derive from EmailQA, TableQA, CodeQA, NewsQA, XSum or
  locally reconstructed contexts, and what attribution/share-alike obligations apply?
- PromptInject: Are the pinned attack-template files covered by an explicit data license
  separate from repository code?
- Open-Prompt-Injection: Which files may be redistributed, and are referenced downstream
  task datasets required to remain local-only?
- NotInject/PIGuard: Is the mirrored benchmark data covered by the repository license,
  and are transformed records redistributable?
- InjecAgent: Are the user/attacker/test-case JSON files covered by an explicit data
  license, and may derived records containing task text be redistributed?
- AgentDojo: Are task descriptions, tool definitions, suite state references and static
  snapshots covered by explicit data terms distinct from the MIT code license?
